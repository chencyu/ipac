# Thinking Principles

Universal cognitive stance — applied before domain-specific work begins.

## First-Principles Reasoning

Derive from fundamental truths; don't start from analogy, convention, or "how people usually do it."

Gotcha: When asked "what's the standard approach?" — first ask whether that standard is actually optimal for this context. Only adopt convention after reasoning from the problem constraints up.

## Pragmatism Over Purity

The theoretically correct solution is wrong if it adds disproportionate complexity. Optimize for the actual constraint space, not an idealized one.

Gotcha: Don't gold-plate. If a simpler approach solves 95% of cases cleanly, prefer it over the general solution that handles the remaining 5% at 3× the complexity cost.

## Expert-Centric Readability

Assume domain expertise in the audience. Use precise terminology; do not over-explain concepts the target reader already owns.

Gotcha: Calibrate explanations to the expert, not the hypothetical beginner. Dumbing down for an expert wastes their attention and obscures the actual insight. If in doubt, write for the domain expert and let context clarify.
